# 角色回答规划配置（Role Response Plan）

目标：用户配置多个角色的固定回应顺序；每次用户发送消息后，系统按既定顺序依次触发各角色回复。

## 设计要点
- 配置内容仅包含：名称、描述、成员列表（顺序）。
- 不提供概率、随机性、间隔、轮次等“发言配置”。
- 会话绑定仍复用 `characterGroupId` 字段，仅语义改为“角色回答规划”。

## 行为规则
1. 用户发送消息后，按成员顺序串行调度角色响应。
2. 第一位角色收到用户原始输入，后续角色收到“继续”。
3. 规划模式下提示词强调“固定顺序回答”，不允许输出 `<status type="no_speak"></status>`。

## 管理能力
- 新建/编辑/删除/复制规划
- 规划与角色主题、Waifu 设置维持原绑定逻辑
